Privacy Policy

We don't spy, we just play music (and count your achievements)

TL;DR: we store minimum data, sell nothing, don't track you. Bot is for music and fun, not for data collection.

🆕 v4.7 Update: Bot now stores listening statistics for achievements system. All data is local, anonymized and used only for your /stats and /top.

What We Collect (and why)

Discord Server ID
To know which queue plays where. Deleted on bot restart.
Channel and Message IDs
To update control panel (progress bar, buttons). Only for current session.
Queue History (20 tracks)
For ⏮️ (previous track) and 📜 (history) buttons. Stored in RAM, not database.
Listening Statistics (v4.7+)
Track count, total time, genres, achievements. Stored in local SQLite. Required for /stats, /top, /globaltop and 30 achievements system.
Discord User ID (v4.7+)
To bind statistics to your account. Stored as hash/ID, no email/phone. Used only for displaying your personal achievements.

What We DO NOT Collect

  • Emails, passwords, personal messages, phone numbers
  • Listening history beyond statistics (specific tracks not saved)
  • IP addresses or geolocation
  • Cookies (site is static, no trackers)
  • Search queries (temporary only for search, not stored)

SQLite Database (v4.7)

Starting from version 4.7, bot uses local SQLite database to store:

  • user_stats: your tracks, time, genres, achievements, first join
  • server_stats: activity by servers (for /top)
  • global_tracks: track popularity counters (for /globaltop)
  • guild_settings: autopilot on/off, language preferences

Database is stored locally on bot server, not in cloud, not transferred to third parties. When bot is removed from server — server stats deleted, personal remains (bound to your Discord ID).

DonationAlerts

When donating via DonationAlerts we see:

  • Your name (as specified in payment)
  • Amount and message (if you wrote one)
  • This data is displayed on fund page and in Discord channel

Want anonymity? Write "anonymous" in donation message or contact us — we'll remove from list.

Security

Tokens and API keys are stored in secure environment variables (.env), not in code. Bot code is private, repository is public only for static files (website).

SQLite database is protected by filesystem access rights. Backups are made by developer locally, not in cloud services.

Your Rights (GDPR/CCPA and common sense)

You can at any time:

  • Remove bot from server — all server data immediately deleted from memory
  • Request donation info removal from public list
  • Request complete statistics deletion from database (contact @jsickwell)
  • Learn what data exists about you (though there's almost none)
